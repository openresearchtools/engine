# AGENTS Instructions

## Build and Test Artifact Policy

- All build artifacts, temporary outputs, and generated binaries are allowed **only** in the sibling directory `../ENGINEbuilds`.
- No build or test artifacts are allowed inside this repository directory (`.`), including but not limited to:
  - `target/`
  - `build/`
  - `bin/`
  - `obj/`
  - `out/`
  - log artifacts
  - generated or downloaded model caches
  - temporary test outputs
- Do not add, leave, or commit any files generated by compilation, testing, packaging, downloads, or benchmark runs inside this repo.
- Always configure toolchains, CI, and local scripts to write generated artifacts to `../ENGINEbuilds` outisde the repo in the parent folder.

## Operational Note

- Keep repository changes limited to source, configuration, and documentation updates.
- If a workflow needs a place for outputs, use `../ENGINEbuilds/<environment>/<purpose>/` so the repo remains clean and reproducible and source code only.
- Build scripts themselves can be inside `build/` folder in the repo, but no artifacts.
